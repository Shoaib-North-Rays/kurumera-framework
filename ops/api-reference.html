Developer reference

The Storefront API

Every customer-facing endpoint your theme can call — the typed @kurumera/storefront SDK, plus the full REST surface for everything beyond it. All read-only-by-default, tenant-scoped, and safe to call from the browser.

01 · Setup

Getting started

Install the SDK (it's already a dependency in a scaffolded theme) and create a client. The client resolves the store for you — from a read-only ksf_… token, or a tenant slug, or a verified domain.

lib/kurumera.ts — server
import { createKurumeraClient } from "@kurumera/storefront";

const kurumera = createKurumeraClient({ token: process.env.KURUMERA_STOREFRONT_TOKEN });
const { results } = await kurumera.products.list({ limit: 12 });
client-side (browser) — resolve by store slug
// On <slug>.kurumera.com the browser resolves the slug from the host.
const kurumera = createKurumeraClient({ tenant: "my-store" });
const cart = await kurumera.cart.create();
  • Base URL — https://admin.kurumera.com/api/v1 (override with apiUrl).
  • Tenant header — the client sends X-Kurumera-Storefront-Token, X-Tenant-ID, or X-Tenant-Domain automatically.
  • Errors — non-2xx throws KurumeraError with .status and .body.
  • Escape hatch — kurumera.http.get/post/patch/del reaches any endpoint the typed resources don't cover.
02 · Typed SDK

SDK resources

The recommended surface — fully typed, one method per common need. Each maps to a REST endpoint (shown on the right).

kurumera.products

GET
.list(params?)Paged product cards. Params: limit, offset, page, collection, vendor, tag, sort.→ /storefront/products/
GET
.getByHandle(handle)Full product detail — variants, options, images, price, SEO.→ /storefront/products/{handle}/
GET
.bestSellers(params?)Top-selling products.→ /storefront/best-sellers/
GET
.deals(params?)Products on sale.→ /storefront/deals/

kurumera.collections

GET
.list(params?)All collections (categories).→ /storefront/collections/
GET
.getByHandle(handle)A collection plus its products.→ /storefront/collections/{handle}/
GET
.getBySlot(slot)Merchandising-slot lookup (e.g. "homepage-featured").→ /storefront/collections/by-slot/{slot}/

kurumera.search

GET
.query(q, params?)Full search — returns product cards.→ /storefront/search/?q=
GET
.autocomplete(q)Type-ahead suggestions.→ /storefront/search/autocomplete/?q=

kurumera.cart

POST
.create()Start a cart; returns its token — persist it (cookie/localStorage).→ /cart/
GET
.get(token)Fetch a cart with its lines and totals.→ /cart/{token}/
POST
.addLine(token, {variant_id, quantity})Add a variant to the cart.→ /cart/{token}/lines/
PATCH
.updateLine(token, lineId, {quantity})Change a line's quantity.→ /cart/{token}/lines/{id}/
DEL
.removeLine(token, lineId)Remove a line.→ /cart/{token}/lines/{id}/

kurumera.navigation · pages · config

GET
navigation.all()Every menu, keyed by handle, hrefs resolved.→ /storefront/menus/
GET
navigation.get(handle)One menu by handle (e.g. "main-menu", "footer").→ /storefront/menus/
GET
pages.getByHandle(handle)A CMS page (about, contact, policies).→ /storefront/pages/{handle}/
GET
config.get()Store branding, colors, contact, SEO defaults.→ /storefront/tenant-config/

The escape hatch — anything else

For endpoints without a typed method (checkout, account, reviews, blogs…), call them directly through kurumera.http — it carries the tenant + auth headers for you.

example — subscribe to the newsletter
await kurumera.http.post("/storefront/newsletter/subscribe/", { email });

// GET with query params
const reviews = await kurumera.http.get(`/storefront/products/${handle}/reviews/`);

04 · Access

Auth model

public

Catalog reads, cart, guest checkout, order create/track, newsletter, contact, waitlist. No user identity needed — the store is resolved from the storefront token, X-Tenant-ID, or the request host. Safe to call from the browser (the SDK client handles it).

customer session

Everything under /storefront/account/*. Call account/login/ to get a customer JWT, then send it as Authorization: Bearer <jwt> on subsequent account requests. This is a customer credential — never an admin one; it can only reach that shopper's own data.

  • Never ship admin credentials in a theme. Themes only ever use the public storefront surface + a customer session the shopper creates.
  • Storefront token (ksf_…) is read-only + tenant-locked — safe to expose in a client bundle (like Shopify's Storefront Access Token). Mint it in the dashboard → Themes → Storefront tokens.
