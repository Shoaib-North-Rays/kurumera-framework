Storefront API — models, endpoints & schemas

Every customer-facing endpoint, with typed request/response schemas grounded in the backend serializers. Read-only-by-default, tenant-scoped, browser-safe. Base URL https://admin.kurumera.com/api/v1.

00 · Conventions

How the API behaves

TopicDetail
Base URLhttps://admin.kurumera.com/api/v1
Store resolutionOne header identifies the store: X-Kurumera-Storefront-Token (a read-only ksf_… token) · X-Tenant-ID (store slug) · X-Tenant-Domain (verified custom domain). The SDK sends the right one for you.
Customer authAccount endpoints need a customer JWT from account/login/ as Authorization: Bearer <jwt>. Everything else is public.
MoneyAlways a decimal string — "1299.00", never a number. Format at the edge with Intl.NumberFormat + the store currency.
IDsUUID strings. Timestamps are ISO-8601 ("2026-07-16T13:12:04.749Z").
PaginationList endpoints return a DRF page envelope: { count, next, previous, results[] }. Params: limit, offset, page.
ErrorsNon-2xx returns { "error": { "code", "message" } } with a matching HTTP status. The SDK throws KurumeraError with .status + .body.
CORSStorefront + cart endpoints allow cross-origin calls, so a theme can call them from the browser.

04 · Access

Auth & errors

public

Catalog, cart, guest checkout, order create/track, content, newsletter, contact, waitlist. The store is resolved from the storefront token, X-Tenant-ID, or the request host — safe from the browser.

customer session

Everything under /storefront/account/* needs a customer JWT (from login) as Authorization: Bearer. It only reaches that shopper's own data — never an admin surface.

Error shape

HTTP 404
{ "error": { "code": "tenant_not_found", "message": "No tenant resolved for this host." } }
  • Storefront token (ksf_…) is read-only + tenant-locked — safe in a client bundle (like Shopify's Storefront Access Token).
  • Never ship admin credentials in a theme. Themes use only the public surface + a shopper's own session.
  • Common codes: 400 validation, 401 missing/expired session, 404 not found / unresolved store, 409 conflict (e.g. out of stock).
